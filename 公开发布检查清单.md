### 分区表

当前 16MB Flash 分区表如下：

```csv
nvs,        data, nvs,      0x9000,    0xC000
otadata,    data, ota,      0x15000,   0x2000
phy_init,   data, phy,      0x17000,   0x1000
factory,    app,  factory,  0x20000,   0x300000
ota_0,      app,  ota_0,    0x320000,  0x300000
ota_1,      app,  ota_1,    0x620000,  0x300000
coredump,   data, coredump, 0x920000,  0x20000
fontfs,     data, spiffs,   0x940000,  0x2E0000
spiffs,     data, spiffs,   0xC20000,  0x3E0000
```

- `0x920000` 是 coredump 分区。
- `fontfs.bin` 必须合并到 `0x940000`。
- 用户 SPIFFS 位于 `0xC20000`，用于图片、画布素材和运行时文件。

### Web 与接口

- HTTP 注册表为 73 条路由。
- Web 嵌入资源为 12 个 HTML 页面和 1 个 PNG 品牌资源。
- 页面入口包括：`/gallery`、`/weather`、`/clock`、`/calendar`、`/message`、`/codex`、`/timetable`、`/todo`、`/countdown`、`/board`、`/config`。
- 天气和 Codex 额度相关配置不在 GET 接口返回明文 Key；保存接口通过 Basic Auth 保护。

### 字库与授权

- `main/font_data.h` 使用开源字体生成，当前公开口径见 `docs/fonts.md`。
- `fontfs` 外置字库位于独立分区，支持 16/24/32px 文字渲染。
- 固件内置或随 release 分发的字体资源不得使用 Windows 系统字体派生产物。
- Web 留言/画板可以使用浏览器临时渲染用户本机字体，该路径不等同于固件分发字体。

### 隐私与仓库卫生

- 仓库不应包含本地构建缓存、临时目录、微信临时文件、本机绝对路径或真实 API Key。
- 默认 AP 密码 `12345678` 属于公开使用说明的一部分，保留。
- 文档中的 `COM3` 是通用串口示例，实际刷机时应替换为用户设备对应端口。

## 剩余验证建议

- OTA 成功、失败和回滚流程仍建议在发版前实机复测。
- 低功耗定时唤醒、天气弱网、轮播长时间运行、连续上传抢占和 5.83 寸屏幕边缘显示效果仍建议按批次抽检。
- 新增或替换屏幕驱动时，应先验证清屏、纯黑、纯红、边框、棋盘格和文字基线测试图，再接入正式页面。
