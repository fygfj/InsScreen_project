# v2.3.3 优化改动说明

本文面向复刻用户和二次开发者，概述 v2.3.3 公开版本中已经合入的主要优化。更细的工程实现请参考 `ENGINEERING_DOCS.md`、`IMPLEMENTATION.md` 和 `docs/fonts.md`。

## 版本定位

v2.3.3 重点收敛显示准确性、低功耗策略、字体显示、5.83 寸适配、Web 页面拆分和发布一致性。默认刷新策略仍以稳定全刷为主，不启用实验性的局部刷新或快刷路径。

## 主要改动

### 日历与时间

- 优化农历和 24 节气计算，补充 2025-2027 年春节、中秋、端午、清明、冬至、小寒、大暑等测试用例。
- 日历显示统一使用已同步的本地时间；时间无效时降级显示，不把错误系统时间当作准确日期。
- 日历模式支持跨日刷新，低功耗定时唤醒时如果保存模式为日历，会在新日期到来后刷新当天页面。

### 低功耗与 WiFi 策略

- 正常上电、复位和按键唤醒保持 AP+STA、mDNS 和 HTTP 后台，方便继续访问网页管理端。
- 只有 RTC 定时器唤醒进入 quick-refresh 路径，并按当前显示模式决定是否启用 STA-only WiFi。
- 图片、轮播、本地待办、本地倒计时和课程表等离线内容默认跳过 WiFi；天气、Codex 额度、带天气摘要或时间无效需要校时的页面才联网刷新。

### 字库与显示

- 内置紧凑字库和外置 `fontfs` 字库统一到可再分发的开源字体来源，避免公开固件依赖系统字体派生产物。
- 4.2 寸页面的标题、正文、小字、普通数字优先走统一字体入口，减少硬放大造成的割裂、发虚和压线。
- 留言板和画板支持 Web 端字体渲染：浏览器把选定字体渲染成图片后上传，适合测试宋体、仿宋、圆体、书法类等本机可用字体，而不需要全部内置到固件。

### 5.83 寸与显示页面

- 增加 648×480 屏幕布局档位，避免简单沿用 4.2 寸坐标或整体放大。
- 天气、时钟、日历、待办、倒计时等页面按 5.83 寸重新调整间距、字号和元素位置。
- 对兼容微雪 5.83b V2 的黑白红屏，配置页优先选择“微雪 5.83 寸黑白红 B V2”；若画面铺满且颜色正常，边缘模糊优先排查屏体、FPC、供电和温度。

### 图片与画板

- 图片黑白模式增加纯黑白/轻量 Bayer 等更适合文字、二维码、线稿和截图的处理路径。
- 固件侧识别预处理过的黑白图像时直接转换黑白平面，并清空红/黄平面，降低重复抖动导致的异常线条。
- 画板布局提交增加防抖和临时文件校验，减少频繁写入对 SPIFFS 的压力。

### 分区与 release

- 分区表调整为 3MB app 三槽、128KB coredump、2.875MB fontfs、3.875MB 用户 SPIFFS。
- `fontfs.bin` 的合并地址为 `0x940000`，`0x920000` 保留给 coredump。
- `release/epaper_uploader_full_16MB.bin` 和 ZIP 已按 v2.3.3 重新生成，并在 `release/README.md` 中记录大小与 SHA256。
- release 固件按普通 Git 文件保留，便于用户从 Gitee 直接下载。

### Web 管理端

- Web 页面从单一配置页拆分出画廊、天气、时钟、日历、留言、额度、课程表、待办、倒计时、画板等入口。
- 配置页存储空间显示改为更明确的“已用/总量/剩余”口径，避免用户误解为只剩少量空间。
- 留言和画板页面加强实时预览、字体选择和操作区域布局，减少滚动时看不到效果的问题。

## 发布注意事项

- 后续如果修改源码、分区表或 fontfs 字库，需要重新执行完整构建并更新 release 固件与校验值。
- 不应提交 `build/`、`managed_components/`、`sdkconfig`、`spiffs_image/`、缓存目录和本地参考目录。
- 固件内置字体资源应继续使用可再分发的开源字体；网页端临时渲染用户本机字体不等同于固件内置分发。
