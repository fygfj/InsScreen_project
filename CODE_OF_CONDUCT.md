# 社区行为规范

本项目面向个人复刻、硬件适配和二次开发。为了让 Issue、PR、交流群和文档讨论保持高效，参与者需要遵守以下协作规则。

## 基本原则

- 尊重不同经验水平的使用者和贡献者，讨论问题时聚焦现象、证据和可复现步骤。
- 提交反馈时尽量给出硬件型号、屏幕型号、固件版本、提交号、串口日志、截图或实机照片。
- 对屏幕驱动、低功耗、字库、图片转换和 Web 页面等问题，优先描述实际硬件环境，不把不同批次屏幕的差异简单归结为代码错误。
- 不公开粘贴真实 WiFi 密码、天气 API Key、Codex/中转站 API Key、私钥、证书、精确住址或其它敏感信息。
- 尊重本仓库的非商业许可边界。涉及量产、销售、商用集成或再授权时，请先阅读 `LICENSE` 与 `COMMERCIAL_LICENSE.md`。

## 不接受的行为

- 人身攻击、辱骂、骚扰、歧视性表达或持续挑衅。
- 故意发布误导性信息、伪造测试结果或隐藏会影响判断的硬件/固件条件。
- 在公开 Issue、PR、评论或截图中泄露他人隐私、密钥、账号、联系方式或未授权的商业资料。
- 将第三方商业字体、闭源素材、未经授权的代码或供应商保密资料直接提交到仓库。
- 反复提交与主题无关、无法复现且拒绝补充信息的问题，影响维护者和其他用户排障。

## 提问与反馈建议

提交问题前建议先阅读：

- `README.md`：项目入口、硬件要求、烧录和首次配置。
- `使用说明书.md`：用户侧网页配置、按键、低功耗和常见问题。
- `SUPPORT.md`：支持渠道、提问前检查和推荐提供的信息。
- `SECURITY.md`：安全问题报告和密钥脱敏建议。
- `.gitee/ISSUE_TEMPLATE.zh-CN.md`：Issue 信息模板。

如果是屏幕显示问题，请尽量提供屏幕尺寸、驱动 IC、颜色类型、配置页选择的面板、BUSY 空闲电平、刷新耗时和屏幕照片。
如果是固件或 Web 问题，请尽量提供固件版本、提交号、浏览器、串口日志和复现步骤。

## 维护者处理方式

维护者可以对不符合规范的内容采取以下处理：

- 要求补充日志、照片、硬件信息或复现步骤。
- 编辑或要求删除包含密钥、隐私、本机路径、商业敏感信息的公开内容。
- 关闭长期无法复现、信息不足或明显超出项目范围的问题。
- 对持续破坏讨论秩序的账号，限制其继续参与 Issue、PR 或交流群讨论。

维护者处理争议时应尽量给出明确原因，并优先保护公开仓库的安全性、可复现性和协作效率。
