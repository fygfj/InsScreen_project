# 喵哎-MiaooAim 开发路线图

## 当前状态（v2.3.4 / 2026-06-28）

喵哎-MiaooAim 当前已经从功能快速迭代阶段进入稳定性、复刻体验和多屏适配阶段。仓库中的源码、Web 管理端、分区表、字库说明、公开发布检查清单和 release 固件已经按 v2.3.4 同步。

当前版本以 **ESP32-S3 + 16MB Flash + 三色墨水屏** 为主要目标，主测屏幕为：

| 状态 | 屏幕 | 说明 |
|------|------|------|
| 主测 | 4.2" SSD1619 BWR，400×300 | 当前默认路径，页面适配完整，默认稳定全刷 |
| 主测 | 5.83" UC8179 / 微雪 5.83" BWR B V2，648×480 | 当前大屏路径，已做页面适配，默认稳定全刷 |
| 实验入口 | ATC/Solum 与 EPD-nRF5 参考面板 | 用于驱动移植和实验验证，未标注主测的面板需自行验证 |

默认刷新策略继续采用稳定全刷。局部刷新、快刷和特殊 LUT 路径不作为当前上线功能开放。

## 当前发布边界

| 项目 | 当前策略 |
|------|----------|
| Flash | 默认分区面向 16MB Flash，不直接支持 8MB Flash |
| PSRAM | N16R8 推荐；N16 可作为降级目标，但大图和复杂画布需要控制内存峰值 |
| app 分区 | factory + OTA0 + OTA1，各 3MB |
| coredump | `0x920000` 起，128KB |
| fontfs | `0x940000` 起，2.875MB，固件随附字库分区 |
| 用户 SPIFFS | `0xC20000` 起，3.875MB，保存图片、画布素材和运行时文件 |
| release | `release/epaper_uploader_full_16MB.bin` 与 ZIP 已按 v2.3.4 重新构建并记录 SHA256 |

后续只要修改源码、分区表、`fontfs` 字库或构建配置，就需要重新执行完整构建，并同步更新 release 固件和校验值。

## 近期优先级

### P0：发布质量与回归验证

- OTA 成功、失败、回滚流程实机复测。
- 低功耗定时唤醒长稳测试：天气、日历、轮播、待办、倒计时分别验证。
- 连续上传、网页推屏、按键切换、后台天气刷新并发回归。
- 5.83" 与 4.2" 主测屏幕的关键页面截图/实机照片抽检。
- release 固件、`firmware-download` 分支和 README 下载入口保持一致。

### P1：显示质量与屏幕适配

- 继续收集 5.83" 不同批次屏幕的 FPC、驱动 IC、BUSY 电平和边缘显示反馈。
- 为新增面板建立最小验证流程：清屏、纯黑、纯红、边框、棋盘格、文字基线、图片色彩。
- 对实验面板逐个标注验证状态，避免用户把实验入口误认为已主测。
- 保持三色屏默认稳定全刷；只有在实机长期验证通过后，才考虑重新开放某类局刷/快刷能力。

### P2：Web 管理端体验

- 继续统一多页面导航、状态提示、存储空间说明和错误反馈。
- 优化移动端配置体验，减少首次复刻时的误操作。
- 画板/留言页面继续保留浏览器字体渲染路线，减少固件内置字体占用。
- 对天气、日历、课程表、待办、倒计时页面保持真实数据驱动，不做静态演示图替代。

### P3：工程可维护性

- 为关键模块补充更细的主机端或真机回归脚本。
- 将常用发布检查命令沉淀到脚本，减少手工漏项。
- 对文档中的路由数量、分区地址、release 校验、字库来源建立发布前一致性检查。
- 保持公开仓库不提交 `build/`、`managed_components/`、`sdkconfig`、`spiffs_image/`、本地字体缓存和个人临时目录。

## 中长期方向

| 方向 | 说明 |
|------|------|
| 更多屏幕型号 | 按实机验证结果逐步把实验入口升级为已验证入口 |
| 9.7" / 大屏 UI | 在确认面板、驱动和 framebuffer 路径后，再建立独立布局档位 |
| 字体体系 | 继续使用可再分发开源字体；固件只内置必要字号，更多字体通过 Web 渲染到图片 |
| 存储优化 | 在保持 OTA 安全和用户 SPIFFS 容量的前提下评估分区调整 |
| 长稳测试 | 轮播、天气、低功耗、OTA 和 SPIFFS 写入场景按天级运行验证 |
| 开发者体验 | 完善二次开发说明、贡献模板、问题模板和硬件复刻检查表 |

## 暂不作为当前目标

- 不默认启用局部刷新或快刷。
- 不为 8MB Flash 提供默认分区表。
- 不把 Windows 系统字体或授权不清晰的字体文件作为固件分发资源。
- 不把所有浏览器可用字体内置到固件；书法、宋体、仿宋等字体优先通过 Web 渲染为图片推屏。
- 不把实验面板入口描述为已完整验证的用户推荐型号。

## 发布前检查清单

每次正式发布前至少确认：

- `CMakeLists.txt` 中 `PROJECT_VER` 已更新。
- `idf.py build` 在 ESP-IDF v5.5.x 环境通过。
- `test/lunar` 在主机端通过。
- `partitions.csv` 与 README、工程文档、release 说明一致。
- `release/epaper_uploader_full_16MB.bin` 重新合并并更新 SHA256。
- `release/epaper_uploader_full_16MB.zip` 如重新生成，需要更新 SHA256。
- `firmware-download` 分支中的 bin 与 `master:release/epaper_uploader_full_16MB.bin` 为同一版本。
- `git diff --check` 通过。
- 文档中无本机路径、微信临时目录、真实 API Key 或会话化描述。
