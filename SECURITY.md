# 安全政策

本项目是面向个人复刻和二次开发的 ESP32-S3 墨水屏终端固件，默认运行在用户自己的局域网或设备 AP 热点环境中。当前安全策略以“避免误暴露、避免误刷写、保护配置数据”为主，不等同于面向公网服务的完整安全产品。

## 支持范围

当前公开维护版本：

| 版本 | 状态 |
|------|------|
| v2.3.4 | 当前公开发布版本，接受安全问题反馈 |
| 早期版本 | 建议升级到当前版本后再复现问题 |

## 已有安全边界

- Web 管理端支持 Basic Auth；启用后，配置、推屏、媒体、数据和 OTA 等敏感接口需要授权。
- `POST /ota` 要求已配置管理账号，避免未设置鉴权时被局域网内设备直接刷写。
- 天气和 Codex/中转站相关配置接口不应在 GET 响应中返回明文 Key。
- 配置导出/导入不包含 Basic Auth 密码。
- 日志和文档发布流程会检查常见 API Key、本机路径、微信临时目录和旧版本残留。

## 报告安全问题

如果你发现以下问题，请不要在公开 Issue 中直接贴出真实密钥或完整内网信息：

- 绕过 Basic Auth 即可修改配置、上传图片、触发 OTA 或读取敏感数据。
- API Key、WiFi 密码、Basic Auth 密码在网页、日志、备份文件或接口响应中明文泄露。
- 构造请求可导致设备反复重启、写坏 SPIFFS/NVS 或异常刷写固件。
- release 固件、分区表或下载说明存在会导致用户误刷写的安全风险。

建议通过 Gitee 私信、仓库 Issue 中的脱敏描述，或 README 中列出的维护者联系方式报告。报告时请提供：

- 固件版本或提交号。
- 复现步骤。
- 受影响接口或页面。
- 已脱敏的串口日志、HTTP 请求/响应或截图。
- 影响范围判断，例如是否需要开启 Basic Auth、是否仅局域网可触发、是否会破坏用户数据。

## 用户侧建议

- 首次配网完成后，在配置页设置管理账号和密码。
- 不要把设备 Web 管理端直接暴露到公网。
- 分享日志和截图前，先隐藏 WiFi SSID、局域网 IP、天气 API Key、Codex/中转站 API Key 和个人位置信息。
- 使用他人提供的固件前，优先核对 `release/README.md` 中的 SHA256。

## 发布前检查

维护者发布前应运行：

```powershell
python tools/check_public_docs.py
git diff --check
```

如果源码、分区表、fontfs 或 release 固件变化，还需要重新构建固件、更新 SHA256，并同步 `release/README.md`、`公开发布检查清单.md` 和 `firmware-download` 分支。
