## 首页首屏建议

Gitee README 首屏建议保留以下顺序：

1. 标题、徽章和一句话简介。
2. `一分钟了解项目`：说明这不是单一 Demo，而是固件 + Web + 文档 + release 的完整工程。
3. `快速体验`：把免编译烧录、首次配置、二次开发、硬件复刻、新屏适配放在同一张表。
4. `项目入口`：集中放 Gitee、固件下载、release 说明、嘉立创硬件和文档索引。
5. `当前发布状态`：只写对复刻用户和开发者有用的状态，不写本机临时检查过程。

这样 Gitee 自动目录会优先显示用户最关心的入口，后续的硬件、API、分区和文档导航保留在 README 下半部分。
